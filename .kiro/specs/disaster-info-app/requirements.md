# 要件定義書

## 概要

住所、緯度経度、SUUMOのURL、現在地などを指定することで、その地域の防災情報を一覧化して表示するアプリケーション。ユーザーが不動産情報や居住地の防災リスクを簡単に把握できるようにする。

## 要件

### 要件1

**ユーザーストーリー:** 不動産検討者として、住所を入力することで、その地域の防災情報を一覧で確認したい。そうすることで、住居選択時の判断材料にできる。

#### 受入基準

1. WHEN ユーザーが住所を入力 THEN システムは その住所の防災情報を取得して表示する
2. WHEN 住所が無効または見つからない場合 THEN システムは 適切なエラーメッセージを表示する
3. WHEN 防災情報の取得に失敗した場合 THEN システムは エラー状況を明確に表示する

### 要件2

**ユーザーストーリー:** ユーザーとして、緯度経度を直接指定することで、正確な位置の防災情報を取得したい。そうすることで、より精密な防災リスク評価ができる。

#### 受入基準

1. WHEN ユーザーが緯度経度を入力 THEN システムは その座標の防災情報を取得して表示する
2. WHEN 緯度経度の形式が無効な場合 THEN システムは 入力形式エラーを表示する
3. IF 緯度経度が日本国内の範囲外 THEN システムは 対応範囲外であることを通知する

### 要件3

**ユーザーストーリー:** ユーザーとして、現在地の防災情報を簡単に確認したい。そうすることで、現在いる場所のリスクを即座に把握できる。

#### 受入基準

1. WHEN ユーザーが現在地取得ボタンをクリック THEN システムは 位置情報の許可を求める
2. WHEN 位置情報の許可が得られた場合 THEN システムは 現在地の防災情報を表示する
3. WHEN 位置情報の許可が拒否された場合 THEN システムは 手動入力を促すメッセージを表示する

### 要件4

**ユーザーストーリー:** ユーザーとして、防災情報を分かりやすい形式で一覧表示してほしい。そうすることで、リスクの種類と程度を素早く理解できる。

#### 受入基準

1. WHEN 防災情報が取得された場合 THEN システムは ハザードマップ情報、避難所情報、過去の災害履歴を表示する
2. WHEN 複数の災害リスクが存在する場合 THEN システムは リスクレベル順に整理して表示する
3. WHEN 防災情報が存在しない場合 THEN システムは その旨を明確に表示する

### 要件5

**ユーザーストーリー:** ユーザーとして、防災情報の詳細や出典を確認したい。そうすることで、情報の信頼性を判断できる。

#### 受入基準

1. WHEN 防災情報が表示された場合 THEN システムは 各情報の出典と更新日時を表示する
2. WHEN ユーザーが詳細情報を要求 THEN システムは より詳しい情報や関連リンクを提供する
3. IF 情報が古い場合 THEN システムは 情報の古さについて警告を表示する
